دليل رفع تطبيق Flutter إلى GitHub

خطوة بخطوة لرفع ملفات تطبيق Flutter من مجلد build/web إلى مستودع GitHub

ملاحظة هامة:

قبل البدء، تأكد من:

  • وجود Git مثبت على جهازك
  • امتلاكك حساب على GitHub
  • إنشاء مستودع جديد على GitHub
  • بناء تطبيق Flutter باستخدام flutter build web --release

الخطوات التفصيلية

1

الانتقال إلى مجلد البناء

افتح terminal وانتقل إلى مجلد build/web الخاص بمشروع Flutter

cd build/web
2

تهيئة Git

قم بتهيئة مستودع Git جديد في المجلد

git init
3

إضافة الملفات

أضف جميع الملفات إلى منطقة الانتظار (staging area)

git add .
4

إنشاء commit

أنشئ commit مع رسالة توضيحية

git commit -m "نشر تطبيق Flutter على الويب"
5

ربط المستودع البعيد

اربط المستودع المحلي بمستودع GitHub البعيد

git remote add origin https://github.com/username/repository-name.git
6

رفع الملفات

ارفع الملفات إلى الفرع الرئيسي في GitHub

git push -u origin main

محاكاة عملية الرفع

انقر على الزر أدناه لمحاكاة عملية رفع الملفات إلى GitHub:

$ git init
$ git add .
$ git commit -m "نشر تطبيق Flutter على الويب"
$ git remote add origin https://github.com/username/repository-name.git
$ git push -u origin main

تفعيل GitHub Pages

بعد رفع الملفات، قم بتفعيل GitHub Pages من إعدادات المستودع:

  1. اذهب إلى مستودعك على GitHub
  2. انتقل إلى الإعدادات (Settings)
  3. ابحث عن قسم Pages في القائمة الجانبية
  4. اختر الفرع main كمصدر
  5. احفظ التغييرات
  6. انتظر بضع دقائق حتى يتم نشر التطبيق
